Archlinux基础安装配置

前期准备

镜像下载

官网下载：archlinux.org 「提供全球各地区镜像源」

国内镜像：阿里云、清华源、中科大、浙大源、163源

下载最新 iso 镜像文件，请自行校验文件签名、文件完整性(PGP、MD5)。

U盘刻录

注意！U盘会被格式化，请提前做好U盘数据备份！

Ventoy 官方下载：ventoy.net 、 github

解压并打开 ventoy 如图，设备会自动检测 U 盘；

配置选项 --> 分区类型 --> 选择 GPT；

点击安装，等待安装结束；

然后将下载好的 iso 镜像文件拷贝到 U 盘中。

推荐使用 ventoy 的理由

一次安装无限使用：ventoy 几乎支持所有类型系统安装，因此安装不同系统，无需反复地格式化制作镜像U盘，只需把系统 ios 镜像文件直接拷贝到U盘即可。且完全不影响 U 盘日常使用。

Rufus 官方下载：rufus.ie 、github

打开 Rufus 如图，设备会自动检测到 U 盘；

选择下载好的 iso 镜像文件；

分区类型：有 MBR 和 GPT，推荐 GPT；

目标系统类型：默认随分区类型改变；

其他选项默认，点击开始，等待完成即可。

磁盘分区表格式 GUID(GPT) 和 MBR 区别

磁盘分区表格式 GUID(GPT) 和 MBR 区别：

  • GPT分区表：GUID Partition Table，全局唯一标识分区表，与 MBR 最大 4 个分区表项的限制相比，GPT 对分区数量没有限制，但 Windows 最大仅支持128个 GPT 分区，GPT 可管理硬盘大小达到了 18EB。只有基于 UEFI 平台的主板才支持 GPT 分区引导启动。

  • MBR分区表：Master Boot Record，硬盘主引导记录分区表，只支持容量在 2.1TB 以下的硬盘，超过 2.1TB 的硬盘只能管理 2.1TB，最多只支持 4 个主分区或三个主分区和一个扩展分区，扩展分区下可以有多个逻辑分区。

两种磁盘分区类型，分别对应主板BIOS两种引导启动模式：

  • 磁盘分区格式为 GUID(GPT) 格式，引导启动模式为新的 UEFI。
  graph LR
	1(开机) --> 2[UEFI初始化] --> 3[引导操作系统] --> 4(进入系统)
  • 磁盘分区格式为 MBR 格式，引导启动模式为传统 Legacy；
  graph LR
	1(开机) --> 2[BIOS初始化] --> 3[BIOS自检] --> 4[引导操作系统] --> 5(进入系统)

对比传统 BIOS 引导启动，UEFI 减少了 BIOS 自检的步骤，节省了大量时间，从而加快系统启动。

磁盘准备

系统安装需要准备一块 空闲 的磁盘空间，空闲指的是 未分配，推荐空间 30G以上；

如果是整个磁盘用于安装，注意磁盘数据备份，确保无重要数据；

如果是在磁盘的某个分区安装，确保该分区未分配。

windows 可在磁盘管理中，通过删除卷或压缩卷，分出一块未分配的空闲空间，也可通过 DiskGenius 软件进行分区操作。

磁盘分区

常用分区工具有fdisk、cfdisk、gdisk、cgdisk等，fdisk 命令风格，cfdisk 图形化风格，这里使用fdisk。

首先查看磁盘和分区情况，根据磁盘空间大小，判断要安装的是哪个磁盘；

假如 /dev/sda 就是我们要安装的磁盘，可以通过如下命令查看该磁盘的分区情况：

如果磁盘有多个分区，这里会显示 /dev/sda1、/dev/sda2 等多个分区信息，start 和 end 代表磁盘起始和结束的扇区号，此时我们在该磁盘的未分配空间上开始建立新分区，用来安装 ArchLinux。

进入 fdisk 分区工具，可以使用如下功能：

m：查看帮助
g ：创建新的 GPT 分区表
o ：创建新的 BIOS 分区表
n ：新建分区
p ：查看已分区信息列表
t ：更改分区类型
w：保存本次分区操作并退出
q：不保存本次分区操作并退出

如果是 UEFI 模式，我们创建三个分区：

  • EFI 引导分区(512M)用于系统引导；

  • swap 交换分区 (4G)用于系统休眠处理缓存；

  • 根分区(剩余空间)用于安装系统 。

如果是一块全新的硬盘，首先输入 g，创建新的 GPT 分区表，否则直接执行下面操作；

创建第一个分区（EFI 引导分区）

1、输入 n （新建分区）
2、直接回车选择默认（Partition type 分区类型，p 是主分区，e 是扩展分区）
3、直接回车选择默认（Patition number 分区号）
4、直接回车选择默认（First sector 开始的部分）
5、输入 +512M，然后回车（Last sector 结束的部分）
6、输入 t ，然后回车（更改分区的类型，输入 L 可以查看所有类型）
7、输入 uefi，然后回车（更改分区的类型为 EFI ）

创建第二个分区（swap 交换分区）

1、输入 n （新建分区）
2、直接回车选择默认（Partition type 分区类型，p 是主分区，e 是扩展分区）
3、直接回车选择默认（Patition number 分区号）
4、直接回车选择默认（First sector 开始的部分）
5、输入 +4G，然后回车（Last sector 结束的部分）
6、输入 t ，然后回车（更改分区的类型，输入 L 可以查看所有类型）
7、输入 swap，然后回车（更改分区的类型为 EFI ）

创建第三个分区（根分区）

1、输入 n （新建分区）
2、直接回车选择默认（Partition type 分区类型，p 是主分区，e 是扩展分区）
3、直接回车选择默认（Patition number 分区号）
4、直接回车选择默认（First sector 开始的部分）
5、直接回车选择默认（Last sector 结束的部分，将剩余空间都给到根分区）
6、输入 p，查看分区列表：/dev/sda1 (EFI)、/dev/sda2 (Linux swap)、/dev/sda3 (Linux)
（如果不对，输入q重新来一遍）
7、输入 w，回车退出（将分区表写入磁盘，并退出）

如果是 BIOS 模式，我们创建两个分区：

  • swap 交换分区 (4G)用于系统休眠处理缓存；

  • 根分区(剩余空间)用于安装系统 。

如果你的是一块全新的硬盘，首先输入 o，创建新的 MBR 分区表，否则直接执行下面操作；

创建第一个分区（swap 交换分区）

1、输入 n （新建分区）
2、直接回车选择默认（Partition type 分区类型，p 是主分区，e 是扩展分区）
3、直接回车选择默认（Patition number 分区号）
4、直接回车选择默认（First sector 开始的部分）
5、输入 +4G，然后回车（Last sector 结束的部分）
6、输入 t ，然后回车（更改分区的类型，输入 L 可以查看所有类型）
7、输入 swap，然后回车（更改分区的类型为 EFI ）

创建第二个分区（根分区）

1、输入 n （新建分区）
2、直接回车选择默认（Partition type 分区类型，p 是主分区，e 是扩展分区）
3、直接回车选择默认（Patition number 分区号）
4、直接回车选择默认（First sector 开始的部分）
5、直接回车选择默认（Last sector 结束的部分，将所有空间都给到根分区）
6、输入 p，查看分区列表：/dev/sda1 (Linux swap)、/dev/sda2 (Linux)（如果不对，输入q重新来一遍）
7、输入 w，回车退出（将分区表写入磁盘，并退出）
